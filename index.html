1. 索引原理

​ 所以索引是干啥的，一言以蔽之，索引相当于书的目录，通过目录（索引）可以更快找到书中的对应内容（数据表中的对应数据），创建索引使用了一种特殊的数据结构：B+树。

2. B树和B+树

mysql的myisam和InnoDB引擎都使用B+树作为索引的数据结构

3. 索引的最左匹配原则

​ 在使用索引进行匹配时，从左到右进行匹配，遇到范围查询(> ,< , between, like)时停止

二分查找 ACWING789

  • 二分查找的边界问题
    • 左边界：寻找不小于k的第一个数（小于k的最后一个数）
      • mid指针对应的值小于k则更新l，使l从mid+1开始找起
      • mid指针大于等于k，则更新r为mid，使得当mid对应值等于k的时候r也能保持更新，知道找到左边界为止
    • 右边界：寻找不小于k的最后一个属（大于k的第一个数）
      • mid指针对应的值大于k则更新r，使r从mid-1开始找起
      • mid指针小于等于k则更新l为mid，使得当mid对应的值等于k的时候也能移动l直到找到右边界为止
  • 死循环问题
    • 这种算法下，寻找左边界无需担心该问题，当寻找右边界时，由于递归的出口是l >= r，更新l=mid的时候，若l和r差1，使用mid = l + r >>1可能会导致mid值不变最终使得l指针不更新，进入死循环，因此，寻找右边界需要将mid置为l + r + 1 >> 1

DO,DTO,VO,BO,POJO

发表于 2022-08-08 更新于 2022-08-10 分类于 后端开发

DO、DTO、BO、VO、POJO区别 - 知乎 (zhihu.com)

  • DO:domin object，抽象的数据表对象

  • DAO:data access object，实际操作数据表时，使用dao对数据表进行封装，来实现java业务代码对数据库的操作，包含了对实际数据库操作的各种方法封装

  • DTO:data transfer object，没有对应的数据表，当接口之间需要相应的数据渲染到前台时，使用DTO进行传输，DTO中的字段是DAO的子集

  • VO:view object，渲染到页面上的数据集合，相比于DTO或DAO需要进行数据转换以达到对应的显示格式

  • POJO:plain ordinary java object，只有getter,setter,toString方法的数据包装类，DO,DTO,VO都是POJO，DAO不是
